# Android 온라인 실습 자료 정량 평가

평가일: 2026-07-13

비개발자 동선, 문서 간 중복, 책 15~18장 연결성을 세 관점에서 독립 검토한 뒤 수정 결과를 다시 평가했습니다. 이번 평가는 문서만 대상으로 하며 Android 빌드와 앱 실행 검증은 다시 수행하지 않았습니다.

## 이번에 해결한 핵심 문제

1. 책 17장의 Android 흐름을 `온라인 1~6절에서 점검과 테스트 시작 → 책 17.7절 → 신청 가능 상태가 표시되면 온라인 7절`로 나눴습니다.
2. 책 18장을 접근 승인 전, 출시 미제출, 앱 심사 중, 게시 대기와 정식 공개 상태로 분리했습니다.
3. AAB 업로드 전에 앱 서명 SHA와 Google Play 설치본을 요구하던 순환 절차를 바로잡았습니다.
4. Firebase·App Check·Play Integrity의 프로젝트 검사는 클로드 코드가 맡고, 사람은 콘솔 클릭과 실제 Android폰 확인만 하도록 바꿨습니다.
5. 개인정보처리방침 참고본의 저자 이름과 문의 주소를 독자가 그대로 제출하지 않도록 확인 절차를 추가했습니다.
6. 정식 공개 절차의 단일 원본을 `release-roadmap.md`로 통합했습니다.
7. 직접 모집, 크몽, 개발자 커뮤니티 품앗이는 `tester-recruitment.md`로 분리했습니다.
8. Windows 통합 문서를 독자용 진입점에서 편집·QA 자료로 이동하고 배포 절차 중복을 제거했습니다.
9. 독자용 최종 기술 체크리스트를 삭제하고 결과 확인 중심으로 바꿨습니다.
10. 책용 스크린샷 제작 문장을 독자 실행 본문에서 제거했습니다.
11. Firebase 설정을 첫 AAB보다 먼저 확인하고, App Check 강제 적용 뒤 Google Play 설치본에서 AI 자동 입력을 다시 확인하도록 완료 기준을 추가했습니다.
12. 기본 스토어 등록정보의 카테고리와 지원 연락처, 프로덕션 국가·지역, 앱 심사 중과 수정 요청 경로를 추가했습니다.
13. 내부 테스트와 비공개 테스트의 메시지와 발송 시점을 분리하고, 18.4절에서 사용할 Android 메시지를 추가했습니다.
14. 독자용 시작점에서 QA·TRD·인쇄 문서를 제거하고 루트의 저장소 관리 영역으로 옮겼습니다.
15. 공개 페이지에서 제거한 판본 표기와 이를 다시 요구하던 인쇄 체크 항목을 정리했습니다.
16. 내부 테스트 반영 대기 뒤 참여 링크 생성 여부에 따라 8절 또는 9절로 재개하도록 수정했습니다.
17. Firebase 연결, 내부 테스트 출시, 앱 콘텐츠와 App Check 강제 적용의 완료 상태를 추가했습니다.
18. 비공개 테스트 버전 승격, Google Group 가입, 같은 계정의 내부 테스트 해제 순서를 구체화했습니다.
19. 16장 아이콘 프롬프트와 18장 사용자 표현을 iOS·Android 공통 흐름으로 바꿨습니다.
20. 출판 원본의 개인정보는 편집자가 가리고 비밀번호·인증번호·인증서 지문은 촬영하지 않도록 구분했습니다.
21. 2026년 Android 개발자 인증 흐름에 맞춰 패키지 이름을 앱 생성 시 등록하도록 고치고, 저자나 예제 패키지를 그대로 쓰지 않게 했습니다.
22. 첫 AAB 업로드 전에는 App Check 코드 분기만 준비하고, 업로드 뒤 Play 앱 서명 SHA-256과 Cloud 프로젝트를 연결한 다음 Play Integrity 제공자를 등록하도록 순서를 바로잡았습니다.
23. 에뮬레이터의 debug provider 결과 화면을 Google Play 설치본 검증 예시에서 제거했습니다.
24. 비공개 테스트를 초안 생성에서 끝내지 않고 국가·지역, 출시 노트, 검토, 제출, 제공 상태 확인 뒤 참여 링크를 보내도록 보완했습니다.
25. 프로덕션에는 새 AAB를 바로 올리지 않고 비공개 테스트에서 실제 설치해 확인한 버전만 승격하도록 제한했습니다.
26. 첫 출시는 관리형 게시를 사용할 수 없고 출시 비율 선택도 표시되지 않는다는 공식 흐름을 반영했습니다.
27. 첫 내부 테스트를 48시간으로 단정하던 표현을 없애고 Play Console의 검토·처리 상태와 표시 시간을 따르게 했습니다.
28. 책 17~18장에서 Android가 비공개 테스트·승인 대기 중일 수 있음을 반영해, 공개 앱으로 단정하던 문장을 상태별 표현으로 바꿨습니다.
29. 출판용 원본과 클로드 코드에 공유하는 문제 화면의 개인정보 처리 기준을 분리했습니다.
30. 비공개 테스트의 트랙 만들기·테스터 저장·국가 확인·출시 제출 순서를 추가했습니다.
31. 등록 인원이 아니라 Play Console에 표시되는 실제 opt-in 참여 인원을 기준으로 바꿨습니다.
32. 프로덕션 접근 신청 질문과 마지막 제출 버튼, 검토 중 완료 상태를 추가했습니다.
33. 프로덕션에서 라이브러리로 버전을 추가하는 경로와 비공개 출시 상세에서 승격하는 경로를 분리했습니다.
34. Windows 11장에서 첫 AAB 뒤에만 가능한 Play App Signing 작업을 제거하고 15장으로 이월했습니다.
35. 책 15~18장에 12명·14일 요구사항이 표시되는 계정과 표시되지 않는 계정의 흐름을 함께 반영했습니다.
36. 공개 독자 문서에서 출판사 제출 지시를 제거하고 편집자 문서에만 남겼습니다.
37. 실제 Play Console에서 비공개 테스트 트랙·설정, 프로덕션 접근 조건과 현재 참여 인원, Google Play 보호, Play Integrity 프로젝트, Play 앱 서명 메뉴 원본 6장을 추가했습니다.
38. 2026년 7월 실제 UI에 맞춰 `앱 무결성` 대신 `Google Play로 보호됨`에서 Play 앱 서명과 Play Integrity를 여는 경로로 수정했습니다.
39. 17장 첫 방문, 18장 재방문, 추가 테스트·수정 요청의 상태표를 분리해 7절과 8~9절의 적용 시점을 명확히 했습니다.
40. 책 15장의 12명·14일 안내와 책 18장의 Android 재진입 문장을 짧게 정리해 같은 설명을 반복하지 않도록 했습니다.
41. 비공개 테스트 요구사항이 없는 계정은 17장에서 1~5절과 8절을 진행하고, 심사 대기 중에는 15장의 내부 테스트 링크를 사용하도록 책과 온라인 자료를 연결했습니다.
42. 내부 테스트, 비공개 테스트, Google Play 공개 링크를 18장의 세 가지 실제 상태로 분리했습니다.
43. 전체 창 원본을 유지하면서도 지면에서 글자가 작으면 브라우저 배율을 높여 다시 원본 촬영하도록 판독 기준을 추가했습니다.
44. Markdown을 단일 원본으로 빌드하는 독자용 웹 문서를 추가해 GitHub 저장소 화면을 거치지 않게 했습니다.

## 남아 있는 외부 변수

- Google Play Console 메뉴와 심사 요구사항은 바뀔 수 있습니다.
- 계정 확인, 14일 비공개 테스트, 프로덕션 접근 심사에는 독자별 대기 시간이 생깁니다.
- 개인정보처리방침과 데이터 보안 답변은 독자가 실제로 만든 앱의 동작에 따라 달라집니다.
- 앱의 기존 기술 검증 기록은 [`release-verification-2026-07-12.md`](release-verification-2026-07-12.md)에 별도로 보존합니다.
- Firebase App Check 등록 완료, 프로덕션 접근 신청 폼과 실제 프로덕션 심사·게시 화면은 해당 상태가 실제 계정에 나타날 때 편집하지 않은 원본으로 추가 촬영해야 합니다.

## 출판 전 확인

- 책 원고의 Android 분기 문장과 공개 문서의 상태표가 같은지 확인합니다.
- 최종 PDF의 QR이 고정 Android 안내 페이지로 열리는지 확인합니다.
- 책 배치용 Google Play Console 화면은 편집하지 않은 촬영 원본을 출판사에 제출하고, 주소·전화번호·결제 정보는 편집자가 최종 지면에서 처리합니다.
- 온라인 공개용 원본은 주소·전화번호·결제 정보가 화면에 나오지 않는 상태에서 촬영합니다.
- 비밀번호, 인증번호, 인증서 지문 전체와 전체 테스트 참여 링크는 촬영하지 않습니다.
- Markdown 상대 링크와 이미지 파일 존재 여부를 확인합니다.
