## Google Play Console 화면

출판사 보관·책 배치용 화면은 자르기·리샘플링·가림·합성·주석을 하지 않은 촬영 원본을 전달합니다. 계정 이메일, 다른 앱 이름, 주소와 결제 정보가 보여도 원본 파일은 그대로 전달하고, 편집자가 최종 지면에서 공개하면 안 되는 개인정보를 가립니다.

GitHub 온라인 자료에 넣는 화면도 촬영 원본만 사용합니다. 이메일과 다른 앱 이름은 원본 그대로 사용할 수 있지만, 주소·전화번호·결제 정보는 편집 없이 공개되므로 처음부터 해당 정보가 화면에 나오지 않는 상태에서 촬영한 원본만 사용합니다. 클로드 코드에게 전달하는 화면에는 이메일·주소·전화번호·결제 정보를 포함하지 않습니다. 비밀번호, 인증번호, 인증서 지문 전체와 비공개 테스트 참여 링크 전체는 책·온라인·클로드 코드 전달용 어느 화면에도 포함하지 않습니다.

Play Console 20장은 2026년 7월 13일 전체 브라우저 화면으로 다시 캡처했습니다. 편집용 번호는 파일명의 빈 번호와 무관하게 P-1부터 연속으로 사용합니다.

최종 PDF에서 핵심 버튼과 상태 문장이 읽히지 않는 화면은 기존 파일을 확대하거나 자르지 않습니다. 같은 화면을 브라우저 배율 125%·150%·175% 중 읽기 좋은 값으로 다시 촬영하고, 촬영 뒤 편집하지 않은 새 원본을 배치합니다. 화면별 판독 여부는 실제 인쇄 크기 100%에서 확인합니다.

다음 화면은 현재 계정 상태에서 열 수 없어 원본 촬영을 기다리는 항목입니다. 임의 화면이나 편집본으로 대체하지 않고, 해당 상태가 실제 계정에 나타났을 때 촬영합니다.

이 다섯 화면이 최종 지면에 직접 배치되는 구성이라면 촬영 전에는 교정을 완료로 표시하지 않습니다. 본문 설명만으로 구성한다면 위 대체 자료가 해당 단계와 정확히 맞는지 편집자가 확인합니다.

원본과 배치본 모두 빨간 박스, 화살표, 번호를 추가하지 않습니다. 설명이 필요하면 캡션이나 본문으로 처리합니다.

## 출판 전 최종 확인

- [ ] 종이책의 URL과 QR이 위 고정 주소로 열림
- [ ] 최종 PDF에 배치된 QR을 iPhone과 Android 기본 카메라에서 각각 스캔함
- [ ] 15장·16장 복귀 지점과 17장·18장의 대기/승인 분기가 최종 원고와 일치함
- [ ] Android 앱 화면의 사진·식당명·검색어가 한 사례로 이어짐
- [ ] 최종 지면에서 주소, 전화번호와 결제 정보가 가려짐
- [ ] 촬영 원본과 최종 지면에 비밀번호, 인증번호, 인증서 지문과 전체 참여 링크가 없음
- [ ] 출간 직전 공식 문서에서 Target API와 신규 개인 계정 테스트 요구사항을 다시 확인함
- [ ] README와 독자용 Android 문서의 Markdown 로컬 링크 검사를 통과함

QR 규격과 인쇄 크기, 화면 해상도 판정은 [`print-proof-checklist.md`](print-proof-checklist.md)를 사용합니다.
