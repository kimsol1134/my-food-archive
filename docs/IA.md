## 📱 [수정본] 마이 맛집 아카이브 완벽 IA 및 화면 설계서

### 1. 서비스 정보 구조 (Data Structure)
* **저장 매체:** 기기 로컬 저장소 (클라이언트 전용, Hive 또는 Isar)
* **데이터 모델 (`ArchiveItem`)**
    * `id`: 고유 식별자 (UUID)
    * `imagePath`: 사진 로컬 저장 경로
    * `date`: 방문 일자 (EXIF 자동)
    * `location`: 지역명 (EXIF 위치 정보 기반 변환, 수정 가능)
    * `menuName`: 메뉴명 (AI 추출, 수정 가능)
    * `category`: 카테고리 (AI 추출, 수정 가능)
    * `restaurantName`: 식당명 (**필수 수동 입력**)
    * `searchKeyword`: 검색 최적화용 통합 문자열 (시스템 자동 생성)

### 2. 페이지 계층 구조 (Page Hierarchy)
* **1.0 홈 화면 (`HomeScreen`)** * **1.1 상세 보기 화면 (`DetailScreen`)**
* **2.0 기록 추가 및 수정 화면 (`AddEditRecordScreen`)** *(통합 재사용)*
    * *(시스템 액션)*: OS 갤러리 호출 및 권한 요청

### 3. 화면별 상세 설계 및 데이터/사용자 흐름

#### 1.0 홈 화면 (`HomeScreen`)
* **역할:** 저장된 맛집 리스트를 한눈에 보고 검색하는 메인 뷰.
* **상태(State)에 따른 화면 변화:**
    * **Empty State (데이터가 0개일 때):** 화면 중앙에 "아직 저장된 맛집이 없어요. 하단의 + 버튼을 눌러 첫 기록을 남겨보세요!" 텍스트와 일러스트 표시.
    * **Data State (데이터가 있을 때):** `GridView`로 사진 썸네일 바둑판 배열.
* **레이아웃 컴포넌트:**
    * **[Top] 검색바:** 텍스트 입력 시 `searchKeyword` 기반으로 즉각적인(Real-time) 리스트 필터링.
    * **[Body] 갤러리 뷰:** 썸네일 터치 시 👉 `1.1 DetailScreen`으로 이동.
    * **[Bottom Right] FAB (+ 버튼):** * 터치 시 👉 갤러리 접근 권한 확인. 
        * (권한 거부 시): "설정에서 사진 권한을 허용해 주세요" 스낵바 노출.
        * (권한 허용 시): 사진 선택 후 `2.0 AddEditRecordScreen` (생성 모드)으로 이동. (사진 선택을 취소하면 홈 화면에 머무름).

#### 1.1 상세 보기 화면 (`DetailScreen`)
* **역할:** 개별 맛집 기록의 상세 정보 확인 및 관리(수정/삭제).
* **레이아웃 컴포넌트:**
    * **[Top] AppBar:** * 좌측: '뒤로 가기' 버튼.
        * 우측: **'수정(Edit)', '삭제(Delete)' 아이콘 버튼.**
            * '수정' 탭 시 👉 현재 데이터를 들고 `2.0 AddEditRecordScreen` (수정 모드)으로 이동.
            * '삭제' 탭 시 👉 "정말 삭제하시겠습니까?" 확인 팝업(Dialog) 노출 후 로컬 DB에서 삭제하고 홈으로 복귀.
    * **[Body] 콘텐츠 영역:**
        * 원본 사진 크게 보기.
        * 식당명 (가장 크게), 메뉴명, 카테고리, 지역, 날짜를 깔끔한 텍스트로 나열.

#### 2.0 기록 추가 및 수정 화면 (`AddEditRecordScreen`)
* **역할:** AI 분석 결과를 확인하고, 정보를 최종 입력/수정하여 저장하는 화면. (생성과 수정을 하나의 화면에서 처리하여 코드 효율성 극대화).
* **데이터 흐름:**
    * **생성 모드일 때:** 사진 선택 👉 로딩(AI 및 EXIF 추출) 👉 폼에 초기값 할당 👉 식당명 입력 대기.
    * **수정 모드일 때:** 로딩 없이 기존 `ArchiveItem` 데이터를 폼에 그대로 채워둠.
* **레이아웃 컴포넌트:**
    * **[Top] AppBar:** '취소' 버튼 (작성 중 취소 시 이전 화면으로 복귀).
    * **[Overlay] 로딩 인디케이터:** AI 분석 진행 시 화면 터치를 막고 로딩 중임을 표시.
    * **[Body] 정보 입력 폼 (`TextField` 목록):**
        * 선택된 사진 썸네일 미리보기.
        * 식당명 (필수 입력값. 비어있으면 저장 버튼 비활성화).
        * 지역, 메뉴명, 카테고리 (AI/EXIF 값이 틀렸을 때를 대비해 모두 `TextField`로 제공하여 텍스트 수정 가능).
        * 날짜 (읽기 전용).
    * **[Bottom] '저장' 버튼:** 터치 시 `LocalDB`에 Insert(생성) 또는 Update(수정) 처리 후 홈으로 이동.
