## 📱 [프로젝트명: 마이 맛집 아카이브] MVP PRD (빠른 구현 버전)

### 1. 왜 만드는가 (문제 정의)
* **정보의 파편화:** 기본 사진첩에 음식 사진과 일상 사진이 섞여 있어 과거에 방문한 맛집을 다시 찾기 어려움.
* **기억의 한계:** "거기 맛있었는데 이름이 뭐였지?" 하고 검색하려 해도, 사진첩에서는 '지역명+메뉴' 조합으로 내가 먹은 음식을 찾을 수 없음.
* **입력의 번거로움:** 메모장이나 별도 앱에 식당 이름, 메뉴, 날짜를 일일이 타이핑하는 과정이 귀찮아 결국 기록을 포기하게 됨.

### 2. 누가 쓰는가 (타겟 사용자)
* **오직 나 (1인 타겟):** 유지보수와 비용 부담 없이, 내 스마트폰 안에서 빠르고 가볍게 맛집을 아카이빙하고 싶은 개인.

### 3. 핵심 기능 (빠른 구현을 위한 5가지 필수 기능)

* **① 사진 업로드 및 기본 위치 추출 (스크린샷 제외)**
    * 사용자가 스마트폰으로 직접 촬영한 '원본 음식 사진'만 업로드함.
    * 사진에 내장된 EXIF 데이터(메타데이터)를 읽어와 **'촬영 날짜'**와 **'기본 지역 정보(GPS 좌표를 통한 동/구 단위의 텍스트, 예: 연남동)'**를 자동으로 저장함.
* **② Vision AI 자동 태깅 (핵심 자동화)**
    * 업로드된 이미지를 AI(gemini API)가 분석함.
    * 사진만 보고 **'메뉴 이름(예: 크림 파스타, 삼겹살)'**과 **'카테고리(예: 양식, 한식, 카페)'**를 자동으로 텍스트로 변환해 입력폼에 채워줌.
* **③ 최소한의 수동 입력 (식당명)**
    * 장소 매칭 API(지도 연동)를 당장 구현하지 않으므로, **'식당 이름'**은 사용자가 텍스트로 직접 입력함. 
    * (AI가 채워준 메뉴/카테고리와 자동 추출된 지역 정보 아래에 식당 이름만 가볍게 타이핑하고 저장 버튼을 누름.)
* **④ 직관적인 키워드 검색**
    * 메인 화면 상단의 검색창에 "연남동 파스타", "역삼동 한식" 등 저장된 데이터(지역명, 식당명, 메뉴, 카테고리)를 조합해 검색하면 해당 식당 기록만 즉시 필터링됨.
* **⑤ 내 맛집 갤러리 뷰**
    * 저장된 기록들을 음식 사진이 크게 보이는 바둑판(Grid) 형태의 갤러리 뷰로 나열함.
    * 사진을 탭하면 [식당명 / 지역 / 메뉴 / 카테고리 / 날짜]의 상세 정보를 확인할 수 있음.
